**docker-compose.yml** — Based on official n8n-hosting PostgreSQL template:
- Two services: `postgres` (image: postgres:16) and `n8n` (image: docker.n8n.io/n8nio/n8n)
- Both services: `restart: unless-stopped` (per user decision, NOT `always`)
- PostgreSQL health check: `pg_isready -h localhost -U ${POSTGRES_USER} -d ${POSTGRES_DB}` with interval 5s, timeout 5s, retries 10
- n8n `depends_on` postgres with `condition: service_healthy`
- Named volumes: `db_storage` for PostgreSQL data, `n8n_storage` for n8n data
- PostgreSQL volume mount: `./init-data.sh:/docker-entrypoint-initdb.d/init-data.sh`
- n8n volume mount: `n8n_storage:/home/node/.n8n`
- n8n ports: `5678:5678`
- n8n environment variables (all from .env): DB_TYPE=postgresdb, DB_POSTGRESDB_HOST=postgres, DB_POSTGRESDB_PORT=5432, DB_POSTGRESDB_DATABASE, DB_POSTGRESDB_USER (non-root), DB_POSTGRESDB_PASSWORD (non-root), N8N_ENCRYPTION_KEY, GENERIC_TIMEZONE=Asia/Kuala_Lumpur
- Do NOT include N8N_BASIC_AUTH_* variables — these are deprecated in recent n8n versions. n8n now uses owner setup on first visit (built-in user management). This is more secure than basic auth.
- Docker logging driver with max-size 10m and max-file 3 to prevent disk exhaustion

**.env** — Template with placeholder values and comments:

**init-data.sh** — PostgreSQL initialization script (from n8n-hosting repo):
```bash
#!/bin/bash
set -e;

if [ -n "${POSTGRES_NON_ROOT_USER:-}" ] && [ -n "${POSTGRES_NON_ROOT_PASSWORD:-}" ]; then
  psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
    CREATE USER ${POSTGRES_NON_ROOT_USER} WITH PASSWORD '${POSTGRES_NON_ROOT_PASSWORD}';
    GRANT ALL PRIVILEGES ON DATABASE ${POSTGRES_DB} TO ${POSTGRES_NON_ROOT_USER};
    GRANT ALL ON SCHEMA public TO ${POSTGRES_NON_ROOT_USER};
  EOSQL
fi
```
