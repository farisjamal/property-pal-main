Create the Edge Function at `supabase/functions/crypto-service/index.ts`.

**Edge Function implementation using Web Crypto API (AES-GCM):**

Use the modern `Deno.serve()` pattern (NOT the deprecated `serve` from std library). The function must:

1. Read `ENCRYPTION_KEY` from `Deno.env.get('ENCRYPTION_KEY')` — throw if missing
2. Derive a CryptoKey from the secret using `crypto.subtle.importKey('raw', ...)` with AES-GCM, 256-bit
   - Encode the key string to bytes: `new TextEncoder().encode(key.padEnd(32, '0').slice(0, 32))`
   - Import as raw key material with `['encrypt', 'decrypt']` usages
3. Handle CORS preflight (OPTIONS request returns 200 with appropriate headers)
4. Parse JSON body: `{ action: 'encrypt' | 'decrypt' | 'batch_decrypt', data: string | object | object[] }`
5. Validate JWT — Supabase Edge Functions validate JWT by default (verify_jwt: true), so no manual JWT check needed

**Action: encrypt**
- Input: `{ action: 'encrypt', data: 'plaintext string' }`
- Generate random 12-byte IV: `crypto.getRandomValues(new Uint8Array(12))`
- Encrypt with `crypto.subtle.encrypt({ name: 'AES-GCM', iv }, key, encodedData)`
- Return: `{ ciphertext: base64(encrypted), iv: base64(iv) }` as JSON string
- The response is a JSON string that can be stored directly in the TEXT database column

**Action: decrypt**
- Input: `{ action: 'decrypt', data: '{"ciphertext":"...","iv":"..."}' }` (data is a JSON string from the database)
- Parse the JSON string to extract ciphertext and iv
- Decode base64 back to Uint8Array
- Decrypt with `crypto.subtle.decrypt({ name: 'AES-GCM', iv }, key, ciphertext)`
- Return: `{ data: 'plaintext string' }`

**Action: batch_decrypt**
- Input: `{ action: 'batch_decrypt', data: ['{"ciphertext":"...","iv":"..."}', ...] }` (array of JSON strings)
- Decrypt each item in parallel with `Promise.all`
- Return null for items that fail or are null/empty
- Return: `{ data: ['plaintext1', 'plaintext2', null, ...] }`
- Max 50 items per batch (return 400 if exceeded) to stay within 2-second CPU limit

**Error handling:**
- Invalid action → 400 with `{ error: 'Invalid action. Use encrypt, decrypt, or batch_decrypt' }`
- Missing data → 400 with `{ error: 'Missing data field' }`
- Decryption failure → return null for that item (don't crash the whole request)
- Batch too large → 400 with `{ error: 'Batch size exceeds maximum of 50 items' }`

**CORS headers on all responses:**
```
'Content-Type': 'application/json',
'Access-Control-Allow-Origin': '*',
'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type'
```

- verify_jwt: true (requires valid JWT — matches existing auth pattern)

After deployment, the user must set the ENCRYPTION_KEY secret via Supabase dashboard or CLI (see user_setup).

**1. Rewrite `src/utils/security.ts`:**

Remove ALL CryptoJS imports and VITE_ENCRYPTION_KEY references. Replace with async functions that call the Edge Function.

New implementation:
```typescript
import { supabase } from '@/integrations/supabase/client';
import bcrypt from 'bcryptjs';

/**
 * Encrypts sensitive data via server-side Edge Function (AES-256-GCM).
 * The encryption key never leaves the server.
 */
export const encryptData = async (text: string): Promise<string> => {
  if (!text) return '';
  const { data, error } = await supabase.functions.invoke('crypto-service', {
    body: { action: 'encrypt', data: text }
  });
  if (error) {
    console.error('Encryption failed:', error);
    throw new Error('Failed to encrypt data');
  }
  // Return the JSON string containing {ciphertext, iv} for storage in TEXT column
  return JSON.stringify(data);
};

/**
 * Decrypts AES-GCM ciphertext via server-side Edge Function.
 */
export const decryptData = async (ciphertext: string): Promise<string> => {
  if (!ciphertext) return '';
  try {
    const { data, error } = await supabase.functions.invoke('crypto-service', {
      body: { action: 'decrypt', data: ciphertext }
    });
    if (error) {
      console.error('Decryption failed:', error);
      return '';
    }
    return data?.data || '';
  } catch (error) {
    console.error('Decryption failed:', error);
    return '';
  }
};

/**
 * Batch decrypt multiple ciphertext values in a single request.
 * More efficient for profile pages that load multiple encrypted fields.
 */
export const batchDecrypt = async (ciphertexts: (string | null)[]): Promise<(string | null)[]> => {
  const items = ciphertexts.map(c => c || null);
  if (items.every(i => !i)) return items.map(() => null);

  try {
    const { data, error } = await supabase.functions.invoke('crypto-service', {
      body: { action: 'batch_decrypt', data: items }
    });
    if (error) {
      console.error('Batch decryption failed:', error);
      return items.map(() => null);
    }
    return data?.data || items.map(() => null);
  } catch (error) {
    console.error('Batch decryption failed:', error);
    return items.map(() => null);
  }
};

/**
 * Hashes a Security PIN using Bcrypt.
 * Uses a salt cost of 10.
 */
export const hashPin = async (pin: string): Promise<string> => {
  if (!pin) return '';
  const salt = await bcrypt.genSalt(10);
  return bcrypt.hash(pin, salt);
};
```
